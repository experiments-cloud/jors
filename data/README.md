# Output Files

This folder contains representative anonymized output files generated by the MILP solver.

The outputs are included to support reproducibility, validation, and inspection of the computational experiments. All sensitive teacher identifiers were anonymized before publication.

## Folder contents

```text
outputs/
|-- isc_20251_timetable.csv
|-- isc_20251_rooms.csv
|-- isc_20251_teachers.csv
|-- industrial_20251_timetable.csv
|-- industrial_20251_rooms.csv
|-- industrial_20251_teachers.csv
|-- README.md
```

## Output description

| File | Description |
|---|---|
| `isc_20251_timetable.csv` | Generated timetable for the Computer Systems Engineering instance. |
| `isc_20251_rooms.csv` | Room assignment and room usage summary for the Computer Systems Engineering instance. |
| `isc_20251_teachers.csv` | Teacher assignment and workload summary for the Computer Systems Engineering instance. |
| `industrial_20251_timetable.csv` | Generated timetable for the Industrial Engineering instance. |
| `industrial_20251_rooms.csv` | Room assignment and room usage summary for the Industrial Engineering instance. |
| `industrial_20251_teachers.csv` | Teacher assignment and workload summary for the Industrial Engineering instance. |

## File types

The output files are provided in CSV format.

These files can be opened with common data-analysis tools such as:

- Python pandas,
- spreadsheet software,
- database import tools,
- visualization dashboards.

## Timetable files

The timetable files contain the scheduled academic sessions produced by the solver.

These files are useful for checking:

- assigned teacher,
- assigned course,
- assigned group,
- assigned classroom,
- day,
- time slot,
- room type,
- session completeness.

## Room files

The room summary files contain information related to classroom assignment and utilization.

These files are useful for analyzing:

- classroom usage,
- room occupation frequency,
- most frequently used classrooms,
- theoretical classroom usage,
- laboratory usage,
- underused or highly used rooms.

## Teacher files

The teacher summary files contain information about teacher assignments and workload.

These files are useful for analyzing:

- assigned teaching blocks,
- workload distribution,
- teacher participation in the final solution,
- workload concentration among teachers.

## Anonymization

All teacher identifiers in these files were replaced by generic anonymous labels such as:

```text
P000001, P000002, P000003, ...
```

The mapping between the original teacher identifiers and the anonymized identifiers is not included in this repository.

This prevents disclosure of sensitive institutional or personal information while preserving the structure required for computational reproducibility.

## Feasibility checks

The reported solutions were validated with respect to the main hard constraints of the model:

- no classroom overlap,
- no teacher overlap,
- no group overlap,
- classroom capacity compliance,
- required teaching sessions scheduled,
- single-room assignment per course.

Both representative instances produced feasible timetables with zero classroom, teacher, and group overlaps.

## Notes

These output files are representative anonymized results from the computational experiments.

The original institutional outputs are not included in this repository.
