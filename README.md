# Academic Timetabling MILP


This repository provides the source code, anonymized sample instances, representative outputs, and reproduction instructions for a Mixed-Integer Linear Programming (MILP) approach to university course timetabling.

The model was designed to generate feasible academic timetables under institutional constraints such as teacher availability, classroom capacity, non-overlapping schedules, room-type compatibility, institutional room policies, and single-room assignment per course.

## Model overview

The timetabling problem is modeled as a Mixed-Integer Linear Programming problem. The model assigns teachers, courses, groups, classrooms, days, and time slots while satisfying institutional hard constraints.

The main objective is to minimize the teacher-course assignment cost:

```text
minimize Z = sum(a_pm * y_pmg)
```

where:

- `a_pm` represents the assignment cost or affinity penalty between teacher `p` and course `m`.
- `y_pmg` is a binary variable equal to 1 if teacher `p` is assigned to teach course `m` for group `g`.

The model prioritizes feasibility through hard constraints and evaluates academic quality through the objective function value.

## Main constraints

The implementation considers the following constraints:

- Each course-group pair must be assigned to one teacher.
- Required teaching sessions must be scheduled.
- A teacher cannot teach two sessions at the same time.
- A classroom cannot host two sessions at the same time.
- A group cannot attend two sessions at the same time.
- Classroom capacity must be sufficient for the assigned group.
- Theory courses and laboratory courses must be assigned to compatible room types.
- Each course must keep a single assigned classroom when the single-room option is enabled.
- Teacher availability and institutional room sets are respected when provided.
- Strict institutional room policies can be enforced through filtered JSON instances and environment configuration.

## Strict ISC room validation

A strict validation instance is included at:

```text
data/samples/isc_20251_strict_rooms.json
```

This instance restricts the Computer Systems Engineering case for period 2025-I to 22 authorized rooms:

- 13 theory classrooms: `FF1`, `FF2`, `FF3`, `FF4`, `FF5`, `FF6`, `FF7`, `FF8`, `FF9`, `FFA`, `FFB`, `FFC`, `FFD`
- 9 laboratories: `LCC`, `LAM`, `K11`, `K12`, `K13`, `K14`, `L12`, `L13`, `L14`

The strict validation run produced:

| Metric | Value |
|---|---:|
| Total scheduled sessions | 645 |
| Sessions assigned to FF theory classrooms | 389 |
| Sessions assigned to laboratories | 256 |
| Sessions assigned to external rooms | 0 |
| External-room usage | 0.00% |
| Room-time overlaps | 0 |
| Teacher-time overlaps | 0 |
| Group-time overlaps | 0 |
| Single-room-per-course validation | OK |
| Solver used | CPLEX_CMD_SAFE |

This validation confirms that the ISC instance can be solved under a strict institutional room policy without using external classrooms.

The strict instance can be regenerated with:

```bash
python scripts/make_strict_isc_json.py
```

To use the strict validation instance, configure `.env` as follows:

```env
DATOS_JSON=data/samples/isc_20251_strict_rooms.json
EXPORT_PREFIX=outputs/test_smoke_isc_20251

STRICT_ROOM_SET=1
REQUIRE_CAPACITY_FOR_ROOM=1

ISC_THEORY_ROOMS=FF1,FF2,FF3,FF4,FF5,FF6,FF7,FF8,FF9,FFA,FFB,FFC,FFD
ISC_LAB_ROOMS=LCC,LAM,K11,K12,K13,K14,L12,L13,L14
ISC_EXTRA_THEORY_ROOMS=
```

Then run:

```bash
python src/run_solver.py
```

After solving, the generated calendar can be validated with:

```powershell
$cal = Import-Csv .\outputs\test_smoke_isc_20251_calendario.csv

$cal | Group-Object aula,dia,hora | Where-Object {$_.Count -gt 1} | Format-Table Count,Name -AutoSize
$cal | Group-Object profesor,dia,hora | Where-Object {$_.Count -gt 1} | Format-Table Count,Name -AutoSize
$cal | Group-Object grupo,dia,hora | Where-Object {$_.Count -gt 1} | Format-Table Count,Name -AutoSize
```

If no rows are printed, no room, teacher, or group overlaps were detected.

## Installation

Clone the repository:

```bash
git clone https://github.com/experiments-cloud/jors.git
cd jors
```

Create a virtual environment:

```bash
python -m venv .venv
```

Activate the virtual environment on Windows:

```bash
.\.venv\Scripts\activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

## Configuration

Create a local environment file from the template:

```bash
copy .env.example .env
```

Edit `.env` and configure the local CPLEX executable path:

```env
CPLEX_BIN=C:\Program Files\IBM\ILOG\CPLEX_Studio221\cplex\bin\x64_win64\cplex.exe
```

The real `.env` file is intentionally excluded from the repository because it may contain local paths, solver configuration, database credentials, or other environment-specific information.

## Running the solver

To solve an instance, run:

```bash
python src/run_solver.py
```

Depending on the local configuration, the solver reads the instance from the JSON file defined in `.env` and exports the resulting timetable, classroom assignments, teacher assignments, and validation files.

## Running the dashboard

The Streamlit dashboard can be launched with:

The dashboard allows users to inspect:

## Data anonymization

The sample datasets and representative outputs were anonymized before publication. Teacher identifiers were replaced by generic labels such as:

```text
P000001, P000002, P000003, ...
```

The local identifier mapping generated during anonymization is intentionally excluded from this repository.

The anonymization script is provided at:

```text
scripts/anonymize_repository_data.py
```

The original institutional database is not included in this repository.

## Reproducibility notes

The repository is intended to support reproducibility of the computational experiments. However, CPLEX must be installed locally, and users must configure the solver path in `.env`.

The published sample instances are anonymized. Therefore, objective values and assignment structures are preserved for reproducibility, while sensitive identifiers are removed.

For the strict ISC validation workflow, the following sequence can be used:

```bash
python scripts/make_strict_isc_json.py
python src/run_solver.py
streamlit run src/ui/visualizer_app.py
```
